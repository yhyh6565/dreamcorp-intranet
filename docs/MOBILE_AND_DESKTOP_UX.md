# 모바일 vs PC 디자인/UX 차이점 정리 문서

본 문서는 최근 업데이트를 통해 적용된 **모바일 환경 최적화** 사항과, 그에 따라 구분된 **PC(데스크탑) 환경**과의 디자인 및 동작 차이점을 기술합니다.

---

## 1. 메인 대시보드 (Dashboard)

대시보드 위젯들은 모바일의 좁은 폭과 제한된 높이를 고려하여 정보를 재배치하고 크기를 조정했습니다. PC 버전은 기존의 넉넉한 공간감을 유지합니다.

### 1.1. 담당 어둠 (Facility Status Widget)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **레이아웃** | **수평 배치 (Compact)** | **수직 적층 (Stacked)** |
| **코드 위치** | 이름 **우측**에 나란히 표시 <br> `이름 Qterw...` | 이름 **상단**에 별도 표시 |
| **상태 뱃지** | 우측 상단 | 우측 상단 (동일) |
| **날짜 정보** | **우측 상단** 모서리에 작게 표시 (공간 절약) | **카드 하단**에 시계 아이콘과 함께 표시 |
| **타이틀** | `text-sm` (작은 폰트) | `text-sm` (통일감 유지) |

### 1.2. 오늘의 일정 (Daily Schedule Widget)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **메인 텍스트** | **`text-3xl`** (축소) | **`text-4xl`** (강조) |
| **타이틀** | `text-sm` | `text-base` |
| **여백 (Padding)** | 상하 여백 축소 (Compact) | 기본 여백 유지 |
| **설명 문구** | `text-xs` | `text-sm` |

### 1.3. 공지사항 & 쪽지함 (Notice & Message Widgets)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **카드 타이틀** | **`text-sm`** | **`text-lg` (공지) / `text-base` (쪽지)** |
| **아이콘 크기** | 축소됨 | 기본 크기 |

## 2. 주요 페이지 (Major Pages)

### 2.1. 시설 안내도 (Floor Map)
모바일에서는 스크롤과 터치 조작을 고려하여 헤더 높이를 최소화하고 지도 조작 편의성을 높였습니다.

| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **헤더 영역** | 타이틀('시설 안내도')과 설명을 **가로로 한 줄**에 배치 <br> 패딩 축소 (`p-3`) | 타이틀과 설명을 세로로 배치 <br> 넉넉한 패딩 (`p-6`) |
| **층 선택기** | 높이와 패딩을 줄여 슬림하게 표현 | 넉넉한 높이, 큰 버튼 |
| **지도 뷰** | **가로 스크롤** 가능 <br> **화면 맞춤 확대/축소(+/-)** 버튼 제공 | 전체 화면 뷰 (스크롤 불필요) |

### 2.2. 근무 일정표 (Calendar)
좁은 화면에서 월간 달력을 보는 것이 비효율적이므로, 모바일에서는 '리스트 뷰'를 도입했습니다.

| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **보기 방식** | **주간 리스트 뷰 (Weekly List)** <br> 기준일 전후 3일간의 일정만 표시 | **월간 그리드 뷰 (Monthly Grid)** <br> 한 달 전체 달력 표시 |
| **네비게이션** | 월 이동 버튼 숨김 | 월 이동 버튼 표시 |
| **헤더 폰트** | 타이틀 `text-lg`, 뱃지 `text-xs` | 타이틀 `text-xl`, 뱃지 `text-sm` |

### 2.3. 공지사항 목록 (Notice List)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **헤더** | 타이틀 `text-lg`, 검색창 작게 | 타이틀 `text-2xl`, 검색창 기본 |
| **테이블 제목** | 화면 너비를 넘어가면 **'...' (말줄임)** 처리 <br> 가로 스크롤 없음 | 제목 전체 표시 (공간 충분) |
| **테이블 폰트** | `text-xs` ~ `text-sm` | `text-base` |
| **제목 말줄임** | **`max-w-[150px]` + `truncate`** 적용<br>화면 벗어남 방지 | 전체 표시 (`max-w-none`) |

## 3. 모달 / 팝업 (Modals)

### 3.1. 여우상담실 예약 (Fox Counseling)
복잡한 시간표 테이블을 모바일 화면에 담기 위해 극한의 다이어트를 진행했습니다.

| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **테이블 폰트** | **`text-[10px]`** (매우 작게) | `text-sm` |
| **셀 패딩** | 축소됨 (`p-1` ~ `p-2`) | 기본 패딩 |
| **하단 버튼** | **고정 너비 (`w-24`)**로 통일하여 컴팩트함 유지 | 고정 너비 (`w-24`) 유지 |

### 3.2. 별관 방문 신청 (Annex Visit Request)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **버튼 레이아웃** | **수평 배치 (`flex-row`)**<br>취소/신청 버튼이 나란히 위치 | 수평 배치 (기본) |
| **버튼 너비** | **`flex-1` (균등 분할)**<br>터치 영역 확보 및 균형감 | `w-auto` (텍스트 길이에 맞춤) |

---

> **요약:** 모든 변경 사항은 `md:` (min-width: 768px) 브레이크포인트를 기준으로 분기 처리되어 있으며, 모바일에서는 **'정보의 밀도'**와 **'공간 절약'**을 최우선으로, PC에서는 **'가독성'**과 **'시원한 레이아웃'**을 최우선으로 디자인되었습니다.

### 2.4. 담당 어둠 배정 (Shadow Assignment)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **헤더 타이틀** | **`text-base`** (추가 축소) | **`text-xl`** (기본) |
| **헤더 아이콘** | `h-4 w-4` | `h-6 w-6` |
| **설명 문구** | **`text-xs`** (축소) | 기본 크기 |
| **테이블 컬럼** | **Location 숨김** (`hidden`)<br>**Grade 너비 축소** (`w-[50px]`) | 모든 컬럼 표시<br>Grade 너비 `w-[80px]` |
| **이름 컬럼** | **공간 확보** (Location 숨김 효과)<br>긴 이름도 잘리지 않음 | 기본 |

### 2.5. 어둠 관리 매뉴얼 목록 (Manual List)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **헤더** | 타이틀 `text-xl`, 설명 `text-xs` | 타이틀 `text-3xl`, 설명 `text-base` |
| **검색창** | 높이 `h-9`, 폰트 `text-xs` | 높이 `h-10`, 폰트 `text-base` |
| **필터 배치** | **가로 배치 (나란히)** | 가로 배치 (동일) |
| **헤더 여백** | **`space-y-3`** (좁게) | **`space-y-6`** (넓게) |
| **테이블 폰트** | 식별 코드 컬럼 `text-xs` | 식별 코드 컬럼 `text-sm` |
| **컬럼 너비** | **No: `40px`**, **Grade: `50px`**<br>**Code: `100px`** (축소) | No: `60px`, Grade: `80px`<br>Code: `150px` |
| **이름 컬럼** | **공간 확보** (타 컬럼 축소 효과) | 공간 여유로움 |

### 2.6. 어둠 관리 매뉴얼 상세 (Manual Detail)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **액션 버튼**<br>(개정/인쇄) | **페이지 최하단**에 배치 | **헤더 우측 상단**에 배치 |
| **헤더 폰트** | 타이틀 `text-2xl`, Doc No `text-xs` | 타이틀 `text-3xl`, Doc No `text-sm` |
| **정보/목차 레이아웃** | **가로 배치 (2컬럼)**<br>요약과 목차가 나란히 표시됨 | **가로 배치 (2컬럼)**<br>왼쪽(본문) / 오른쪽(목차) 구조 |
| **섹션 높이** | **동일 높이 (`h-full`)**<br>요약/목차 높이가 같게 맞춰짐 | `h-fit` + `sticky` (목차 스크롤 따라오기) |

### 2.7. 백일몽 임직원 복지몰 (Welfare Mall)

| **검색 버튼** | **숨김** (`hidden`) | 표시 |

| **건강 검진** | **버튼 작게** (`px-6 py-3`)<br>패딩/폰트 전반적 축소 | 버튼 크게 (`px-8 py-6`) |
| **탭/그리드 간격** | 모든 수직 간격(`space-y`, `gap-y`) 대폭 축소 | 넓은 간격 유지 |
| **탭 메뉴** | 패딩 `py-2` (슬림) | 패딩 `py-4` |

## 4. 글로벌 UI 요소 (Global UI Elements)

### 4.1. 토스트 알림 (Toast Notifications)
| 구분 | 모바일 (Mobile) | PC (Desktop) |
| :--- | :--- | :--- |
| **표시 위치** | **화면 하단 (`bottom-0`)**<br>손가락 접근성 고려 | **화면 우측 하단** |
| **정렬 방향** | 위로 쌓임 (`flex-col-reverse` 등 조정) | 동일 |
